学与成设计网

Next.js 全栈重构 · 架构设计方案

技术栈：Next.js 16 + TypeScript + Prisma + PostgreSQL + Tailwind CSS 4

文档版本：v1.0 · 2026-05-18

一、项目核心目标

一句话定义：把旧站 (PHP LzCMS) 的 SKP 模型 / 教程 / 材质 / 软件下载四种核心资产，重构为可规模化运营的 Next.js 全栈内容平台。

🎯 业务目标

  • 日均 5000+ 访客的内容消费体验
  • 后台单人每天录入 20+ 条资源
  • 支持会员 / VIP 分层下载权限
  • SEO 可索引所有资源页

🏗️ 技术目标

  • SQL 注入 / XSS / CSRF 全量防御
  • 图片 OSS 化，服务器零存储压力
  • API 响应 < 200ms（SSG + ISR）
  • 批量上传 1 分钟完成 1 条资源

5.2 批量上传表单设计（核心组件）

resource-form.tsx 设计思路

根据参考文档，后台录入一条资源的字段固定。表单组件应支持两种模式：

  1. 单条模式：常规表单，逐个字段填写
  2. 批量模式：上传一个 TXT（含标题+链接+密码）+ 选择图片文件夹 → 自动解析并逐条提交

字段对照：

文档字段Prisma 字段组件
标题ContentItem.title<Input>
封面图Media(type=COVER)<MediaUploader>
下载地址ResourceFile.fileUrl<Input>
下载密码ResourceFile.password<Input>
所需积分ContentItem.requiredPoints<Input type="number">
首页分类Category.id<Select>
内容图片Media(type=CONTENT)<MediaUploader multiple>
排序ContentItem.sortOrder<Input type="number">

六、认证与权限模型

6.1 三端用户体系

角色权限登录方式
访客（未登录） 浏览公开资源、搜索 无需登录
会员（MEMBER） 下载公开+会员资源、收藏、查看历史 邮箱+密码 / 微信扫码
VIP 全部资源可下载、无需积分 继承会员登录
编辑（EDITOR） 新增/编辑内容、上传文件 邮箱+密码+OTP
管理员（ADMIN） 全权限：用户管理、系统设置、删除 邮箱+密码+OTP

6.2 鉴权中间件

// src/middleware.ts
import { NextRequest, NextResponse } from 'next/server';
import { getToken } from 'next-auth/jwt';

export async function middleware(req: NextRequest) {
  const { pathname } = req.nextUrl;

  // 后台路由保护
  if (pathname.startsWith('/admin')) {
    const token = await getToken({ req, secret: process.env.AUTH_SECRET });
    if (!token || (token.role !== 'ADMIN' && token.role !== 'EDITOR')) {
      // 不暴露后台是否存在 → 统一 404 而非 302
      return NextResponse.rewrite(new URL('/404', req.url));
    }
  }

  // 会员路由保护
  if (pathname.startsWith('/member')) {
    const token = await getToken({ req, secret: process.env.AUTH_SECRET });
    if (!token) {
      return NextResponse.redirect(new URL('/login?redirect=' + pathname, req.url));
    }
  }

  // 安全头（全站）
  const response = NextResponse.next();
  response.headers.set('X-Frame-Options', 'SAMEORIGIN');
  response.headers.set('X-Content-Type-Options', 'nosniff');
  response.headers.set('Referrer-Policy', 'strict-origin-when-cross-origin');
  response.headers.set(
    'Content-Security-Policy',
    "default-src 'self'; img-src 'self' data: https:; style-src 'self' 'unsafe-inline'; script-src 'self' 'unsafe-inline'"
  );
  return response;
}

export const config = {
  matcher: ['/admin/:path*', '/member/:path*', '/((?!api|_next/static|_next/image|favicon.ico).*)'],
};

7.2 下载鉴权流程

用户点击「下载」
  ↓
GET /api/resources/[id]/download
  ↓ Session 校验
  ├─ 未登录 → 返回 401，前端弹登录框
  ├─ role=FREE, visibility=MEMBER/VIP → 返回 403，"会员专享"
  ├─ role=MEMBER, visibility=VIP → 返回 403，"VIP专享"
  └─ 权限通过 → 记录 DownloadLog（去重）→ 返回 ResourceFile.fileUrl + password
    → 前端展示网盘链接和提取码（复制按钮）

八、图片 / 文件存储方案

推荐：阿里云 OSS / Cloudflare R2

封面图和内容图不再存服务器本地 /uploads/，而是：

  1. 前端直传 OSS：用 STS 临时凭证，图片从浏览器直接上传到 OSS，不经过 Next.js 服务器
  2. 回调写入数据库：OSS 上传成功回调 → API 写入 Media 表
  3. CDN 加速：图片 URL 走 CDN，支持缩略图参数（?x-oss-process=image/resize,w_320）
  4. 私有资源保护：会员/付费资源打上 OSS 私有标记，通过 API 签名 URL 鉴权访问
// Media Uploader 组件流程
1. 用户选择文件 → 前端校验格式/大小（< 10MB，jpg/png/webp）
2. fetch POST /api/media/upload-token → 获取 OSS STS 临时凭证
3. 前端用 STS 直传 OSS → 进度条
4. 上传完成 → fetch POST /api/media/confirm { key, contentItemId, type }
5. 服务端写入 Media 表 → 返回 mediaId + CDN URL

十一、技术决策记录

决策选择理由
富文本编辑器 TipTap（不依赖内容的场景用 Markdown） 用户实际并不需要复杂富文本——标题+图片+下载链接即可。简单场景用 textarea + Markdown；如需富文本，TipTap 比 Quill/Plate 更可控。
图片存储 阿里云 OSS 国内访问速度快，STS 直传免服务器中转，成本低
文件上传组件 自研 MediaUploader 需求简单（拖拽+进度条+缩略图），不引入额外依赖
认证库 next-auth (Auth.js) 原生支持 Next.js，内置 CSRF + Session 安全，可接微信/QQ 登录
速率限制 Upstash Redis Ratelimit Serverless 友好，无需运维 Redis
部署 Vercel / 阿里云 ECS + Docker 小流量 Vercel 免费够用；大流量切 ECS
是否需要 CMS 自研后台优先，后续可选 Payload 现有需求用自研表单即可覆盖；Payload 适合扩展到更复杂的 CMS 场景时引入

十二、前端数据流：何时用 Server Component vs Client Component

场景策略原因
首页资源列表Server Component + 直接 Prisma 查询无需交互，服务端直接查库渲染
搜索框输入Client Component + debounce + fetch需要实时响应输入
收藏按钮Client Component + Server Action需要乐观更新 UI
后台表单提交Client Component + fetch POST需要表单状态管理
下载链接获取Server Action网盘密码不应暴露在前端 JS bundle
